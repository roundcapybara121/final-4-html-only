Responsive Redesign

Overview

After looking through Suzzane Collin’s personal website, I tried to identify and analyze flaws in the interface in order to brainstorm possible improvements to the site. I created mockups to visualize the changes I was going to make and then attempted to code those changes into a responsive website.

Usability Issues

These were the usability issues I noticed with the site:

  • Too much scrolling to get information
    • Causes the amount of text to be misaligned on desktop and tablet view
    • The quick links on the right side doesn’t follow you so not easily accessible
    • Endless scrolling on mobile view
    • It’s not clear that some of the text can act as a link . You have to hover over it first to see that it changes color
  • Links doesn’t bring you directly to the product or author page, only brings you to the general site and you’ll have to search for things related to the author yourself
  • The column on the right side contains the same information which can be repetitive with links visible on other tabs

Accessibility Issues

WebAim is a website where you can paste in another website’s URL in which WebAim will access how accessible the website is for users. It will examine properties such as color contrasts, fonts sizes, layout of components and more to determine a score for how accessible a site is.

The website was put into WebAim Wave to examine for accessibility issues and this was the result:

  • Images did not contain alt tags, people who use screen readers will not be able to know that there is an image
  • A heading element was used as padding when it should just be padding that is used
    • Using heading elements can be confusing if the information is empty
  • Image caption text can be really small for some people
  • WebAim detected some contrast issues for the color of the tabs, but it seems to be inaccurate

Reflection on Critiques

After my redesign was peer reviewed, these were the critiques I received:

  • Site felt empty because of the white space on the sides and too much content was taken away.
    • Idea: I would spread the content out more so that there is less white space. Additionally, a patterned background would aid in reducing the emptiness.
  • The scroll issue does not seem to be fixed. Even after opening the collapse button, users still have to scroll through a lot of information
    • Idea: I could put each review into a card that users can click to open. Then it would reveal more information about the review.
  • One person noted that the white text on the orange background was hard to read.
    • Idea: I would have to further research into what colors would be appropriate for a new design. In the future, I will consider colors that contrast better and put that into webAim to reassess.
  • The website was not fully responsive. When viewed on the phone, users felt that the image would block certain parts of the text.
    • Idea: I believe the reason behind this issue is that I did not use the media queries and divs properly. I believe there was an issue with the padding or that I didn’t properly divide the different components into their own divs, allowing them to overlap. To fix this issue, I think I can put the image and the content into a container div, then divide them further into their own, smaller wrappers. Then use display flex and justify content in order to move them from being horizontal to vertical to each other.
  • Users also noted that the design for the tablet was not too different from the desktop version and would like to see something more catered towards tablet users.
    • Idea: I originally designed the tablet interface based off of the desktop interface. After reviewing the aspects of a desktop compared to a tablet, I agree with the critique in that the tablet should be seen as a bigger mobile device. In the future, I will consider the limited use of the keyboard, and need for easy navigation. Additionally, since desktops users have the capability for a more complex design, I will consider making the experience more creative.

Conclusion

This project provided me invaluable insight into the different considerations that designers often make. Though I did not create a design that fully satisfied users, I am glad that I was able to learn how to make these improvements in the future.
